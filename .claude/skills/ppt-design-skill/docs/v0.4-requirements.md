## 2. 第一梯队需求（P0 — 补到 85%）

### 2.1 数据图表 (Data Charts)

#### 2.1.1 背景

当前 Enterprise Pipeline 的 `_populate_slide` 支持 title/bullets/image/cards/diagram/code/exercise，但**不支持数据图表**。然而 python-pptx 已有 `add_chart` API，且 FreeStyle Pipeline 中已有 `ChartBuilder`。关键差距是 Enterprise 流程中没有 `design.get("chart")` 分支。

#### 2.1.2 用户故事

```
作为企业用户
我希望在 content.json 中定义图表数据
系统自动在我的品牌模板上渲染柱状图/折线图/饼图
以便我不必手动在 PowerPoint 中插入图表
```

#### 2.1.3 数据格式 (content.json)

```json
{
  "slides": [
    {
      "goal": "metrics",
      "title": "Q3 Revenue",
      "chart": {
        "type": "bar",
        "title": "季度收入 (万元)",
        "categories": ["Q1", "Q2", "Q3", "Q4"],
        "series": [
          {"name": "2025", "values": [120, 150, 180, 210]},
          {"name": "2026", "values": [140, 170, 220, 260]}
        ],
        "style": {
          "color_scheme": "brand",
          "show_legend": true,
          "show_labels": true,
          "legend_position": "bottom"
        }
      }
    }
  ]
}
```

#### 2.1.4 支持的图表类型

| 类型 | python-pptx 枚举 | 说明 |
|------|------------------|------|
| `bar` | `XL_CHART_TYPE.COLUMN_CLUSTERED` | 簇状柱形图 |
| `bar_stacked` | `XL_CHART_TYPE.COLUMN_STACKED` | 堆积柱形图 |
| `bar_100` | `XL_CHART_TYPE.COLUMN_STACKED_100` | 100% 堆积柱形图 |
| `bar_3d` | `XL_CHART_TYPE.COLUMN_CLUSTERED_3D` | 3D 柱形图 |
| `line` | `XL_CHART_TYPE.LINE` | 折线图 |
| `line_markers` | `XL_CHART_TYPE.LINE_MARKERS` | 带标记折线图 |
| `pie` | `XL_CHART_TYPE.PIE` | 饼图 |
| `pie_3d` | `XL_CHART_TYPE.PIE_3D` | 3D 饼图 |
| `doughnut` | `XL_CHART_TYPE.DOUGHNUT` | 环形图 |
| `area` | `XL_CHART_TYPE.AREA` | 面积图 |
| `scatter` | `XL_CHART_TYPE.XY_SCATTER` | 散点图 |
| `combo` | — | 组合图（柱 + 线），需 XML |

#### 2.1.5 样式系统

- `color_scheme: "brand"` — 使用 BrandSpec 主色/辅色/强调色
- `color_scheme: "auto"` — python-pptx 默认配色
- `color_scheme: ["#2563EB", "#F97316", ...]` — 自定义色板
- 品牌色映射规则：
  - `series[0]` → `primary`
  - `series[1]` → `secondary`
  - `series[2]` → `accent`
  - `series[3+]` → 从 primary 色相旋转生成

#### 2.1.6 接受标准

1. `content.json` 中 `chart` 字段自动渲染对应图表
2. 支持 12 种图表类型（含 fallback 到 bar）
3. 品牌色自动映射到数据系列
4. 图表标题/图例/标签可独立控制显隐
5. 无 chart 数据时不产生任何副作用
6. 图表位置随 density 自动调整大小
7. 通过 `--review` 可预览图表配置

#### 2.1.7 设计约束

- python-pptx `add_chart` 返回 `GraphicFrame`，非 `Shape`，不能用 `shape.fill` 设色
- 系列颜色通过 `series.format.fill.solid()` + `RGBColor` 设置
- `combo` 类型需 XML 混合 `barChart` + `lineChart`，优先级 P2
- 图表数据表 (data table) 通过 `chart.has_data_table` 开关

### 2.2 演讲者备注 (Speaker Notes)

#### 2.2.1 背景

几乎所有正式演示都需要备注。当前系统完全没有备注支持。

#### 2.2.2 用户故事

```
作为演讲者
我希望每页 PPT 自动携带演讲备注
以便我演讲时能在演讲者视图中看到提示
```

#### 2.2.3 数据格式 (content.json)

```json
{
  "slides": [
    {
      "goal": "hook",
      "title": "公司介绍",
      "notes": "强调我们在 AI 领域的技术积累，预计用时 2 分钟。注意不要提到竞品名称。"
    }
  ]
}
```

#### 2.2.4 API

```python
# python-pptx 原生 API，极简
notes_slide = slide.notes_slide
notes_slide.notes_text_frame.text = "演讲备注内容"
```

#### 2.2.5 接受标准

1. `content.json` 中 `notes` 字段自动写入 slide 备注
2. 无 `notes` 时不创建空白备注页
3. 备注支持多段落文本
4. StoryPlanner 自动模式也能生成备注（基于 goal + bullets 生成摘要提示）
5. `--review` 显示备注摘要

#### 2.2.6 设计约束

- 首次访问 `slide.notes_slide` 会自动创建 `NotesSlide`
- 不需要备注时不要调用 `slide.notes_slide`，避免创建空的 notes part
- StoryPlanner 生成备注时用 goal + bullets 合成，不超过 200 字

### 2.3 页码与页脚 (Page Numbers & Footers)

#### 2.3.1 背景

正式文档几乎都需要页码。python-pptx 没有原生 API，需 XML 操作。

#### 2.3.2 用户故事

```
作为企业用户
我希望 PPT 自动添加页码和页脚
以便读者知道当前进度和文档归属
```

#### 2.3.3 配置格式 (brand.json)

```json
{
  "footer": {
    "show_page_number": true,
    "page_number_format": "第 {n} 页 / 共 {total} 页",
    "page_number_position": "bottom_right",
    "show_footer_text": true,
    "footer_text": "Confidential — ACME Corp",
    "footer_position": "bottom_center",
    "font_size_pt": 10,
    "color_role": "muted-foreground"
  }
}
```

#### 2.3.4 位置枚举

| 位置 | X (inches) | Y (inches) |
|------|-----------|-----------|
| `bottom_left` | 0.9 | 7.0 |
| `bottom_center` | 4.667 | 7.0 |
| `bottom_right` | 11.433 | 7.0 |

#### 2.3.5 接受标准

1. 页码在渲染完成后一次性写入（需要知道总数）
2. 支持自定义格式字符串 `{n}` 和 `{total}`
3. 页脚文本支持品牌色映射
4. 封面页 (goal=hook) 默认不显示页码
5. `skip_pages: [1, 2]` 可指定跳过页码的页
6. 无 footer 配置时不添加任何元素

#### 2.3.6 设计约束

- 必须在所有 slide 渲染完成后写入（因为需要 `{total}`）
- 用 `add_textbox` 而非 XML placeholder，确保兼容性
- 封面检测: `goal == "hook"` 或 `slide_index == 0`
- `{total}` 是动态值，需在 `renderer.save()` 之前计算

### 2.4 超链接 (Hyperlinks)

#### 2.4.1 背景

交互式演示和分享型 PPT 常需链接到外部资源或文档内跳转。

#### 2.4.2 用户故事

```
作为演示者
我希望在 PPT 中添加可点击的链接
以便听众能直接访问相关网页或文档内其他页面
```

#### 2.4.3 数据格式 (content.json)

```json
{
  "slides": [
    {
      "goal": "cta",
      "title": "Get Started",
      "bullets": [
        "Visit our website",
        "Contact sales"
      ],
      "links": [
        {"bullet_index": 0, "url": "https://example.com"},
        {"bullet_index": 1, "url": "mailto:sales@example.com"}
      ]
    }
  ]
}
```

也支持独立链接（不绑定 bullet）：

```json
{
  "links": [
    {"text": "Download PDF", "url": "https://example.com/report.pdf", "position": "bottom_right"}
  ]
}
```

#### 2.4.4 API

```python
# python-pptx 原生
run.hyperlink.address = "https://example.com"
# 或 slide 内跳转
run.hyperlink.target_slide = prs.slides[2]
```

#### 2.4.5 接受标准

1. `links` 字段自动为指定文本添加超链接
2. 支持外部 URL (`https://`) 和邮件 (`mailto:`)
3. 支持文档内跳转 (`slide://3` 跳转到第 3 页)
4. 链接文本使用品牌强调色 + 下划线
5. 无 `links` 时不产生任何副作用

#### 2.4.6 设计约束

- `run.hyperlink` 需要 `run` 对象，必须在文本写入后添加
- `target_slide` 是 python-pptx 1.0.2 的实验性 API，需 try/except
- 链接样式：`font.color.rgb = accent_color`, `font.underline = True`

### 3.1 水印 (Watermark)

#### 数据格式 (brand.json)

```json
{
  "watermark": {
    "text": "CONFIDENTIAL",
    "opacity": 0.15,
    "rotation": -45,
    "font_size_pt": 72,
    "color_role": "muted-foreground",
    "position": "center",
    "skip_pages": [1]
  }
}
```

#### 实现方式

- 添加全屏透明 textbox，旋转 -45°
- 透明度通过 `a:solidFill` + `a:alpha` XML 属性控制（python-pptx 无原生 API）
- 封面页默认跳过

#### 难度: 低

---

### 3.2 目录页 (Table of Contents)

#### 数据格式 (content.json)

```json
{
  "meta": {
    "title": "2026 Annual Report",
    "toc": true,
    "toc_style": "numbered"
  }
}
```

#### 实现方式

- Pipeline 在所有 slide 渲染完成后，根据各页 title 生成 TOC 页
- TOC 插入到封面之后（index=1）
- `toc_style: "numbered"` 显示 "1. xxx"，`"dotted"` 显示 "xxx ... 5"
- 需要后处理：渲染完 → 读 titles → 插入 TOC → 重新编号页码

#### 难度: 中

### 3.4 图标库 (Icon Library)

#### 数据格式 (content.json)

```json
{
  "goal": "features",
  "title": "Key Features",
  "bullets": ["Fast", "Secure", "Scalable"],
  "icons": [
    {"bullet_index": 0, "name": "rocket", "size": "medium"},
    {"bullet_index": 1, "name": "shield", "size": "medium"},
    {"bullet_index": 2, "name": "chart-up", "size": "medium"}
  ]
}
```

#### 图标来源

1. **内置 SVG 集** — 打包 `data/icons/` 目录，~200 常用图标 (MIT)
2. **用户自定义** — `brand.json` 中 `icon_dir: "icons/"` 指定项目内图标
3. **API 生成** — 调用 LLM 图像引擎生成简单图标

#### 渲染方式

- SVG → Pillow rasterize → `add_picture` 插入
- 单色图标支持品牌色着色 (Pillow `ImageOps.colorize`)
- 位置: bullet 左侧 0.5" x 0.5" 区域

#### 难度: 中

## 4. 集成设计

### 4.1 _populate_slide 扩展

当前 `_populate_slide` 处理 7 种内容类型，需新增 4 种：

```
title → subtitle → bullets → image → cards → diagram → code → exercise
                                                    ↑ 新增 ↓
                                              chart → notes → links
```

渲染顺序理由：
- **chart** 在 image 之后：图表占据类似区域，避免遮挡
- **notes** 在最后：不影响视觉布局
- **links** 在 bullets 之后：依赖已写入的文本 run

### 4.2 Pipeline 后处理流程

```
渲染所有 slide
    ↓
添加 TOC（如 meta.toc=true）
    ↓
添加水印（如 brand_spec.watermark）
    ↓
添加页码/页脚（需要 total 页数）
    ↓
renderer.save()
```

### 4.3 content_parser 扩展

新增字段透传：`chart` / `notes` / `links`

### 4.4 BrandSpec 扩展

新增字段：`footer` / `watermark`

### 4.5 DiagramEngine 扩展

新增图表类型（可选，P2 优先级）：
- `bar_chart` — 用 Shape 模拟柱状图（不依赖 add_chart）
- `pie_chart` — 用 Shape 模拟饼图

> 注意：真实数据图表（add_chart）走 pipeline 直路，不经过 DiagramEngine。
> DiagramEngine 中的图表是用 Shape 模拟的简化版本，用于设计图示意。

```
  路演/提案  = 20  (最高频)
  商业报告  = 18
  教育/培训 = 15
  营销/产品 = 12
  战略/咨询 = 10
  技术/架构 = 8
  政府/机构 = 7
  个人/创意 = 5
  研究/学术 = 5
```
